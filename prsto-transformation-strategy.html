1. État actuel du site

PRSTO a deux landing pages distinctes avec deux positionnements différents. Problème : dilution du message, confusion utilisateur, friction dans le funnel.

/prsto — Version Candidat

B2C · Existant

Message : "L'OS qui décroche des postes de direction"

  • Hero : candidat cadre dirigeant en recherche active
  • Features : ATS Scanner, CV Optimizer, LinkedIn Optimizer, Market Radar, CRM Recruteur, Interview Studio
  • Pricing : 0€ / 9.90€ semaine → 149.90€/6mois
  • CTA : "Commencer gratuitement" (pour le candidat)
  • Offres one-shot : Executive Brief 29.90€, Panel 14.90€/session

/recruiters — Version Recruteur

B2B · Existant

Message : "Le logiciel que vos candidats méritent"

  • Hero : recruteur qui prépare ses candidats pour ses clients
  • Features : CV Formatter, Lettre Motivation, ATS Scanner, LinkedIn Optimizer, Market Radar, Brief Entretien
  • Pricing : Solo 99$/mois, Pro 199$/mois, Elite 349$/mois
  • CTA : "Découvrir l'offre Pro"
  • Workflow : Ajouter candidat → Coller offre → Générer dossier → Envoyer client
Problème central : Deux pages = deux positionnements = deux funnels = ressources diluées.
Le recruteur arrive sur /prsto (candidat) et repart. Le candidat arrive sur /recruiters (recruteur) et repart.

2. Les 4 options stratégiques

Chaque option a un impact différent sur le produit, le pricing, le branding, et le développement technique.

A. Statu quo

Conserver les deux pages indépendantes

✅ Pour
  • Aucun travail de refonte
  • Chaque audience a sa page
  • Pas de risque de casse
❌ Contre
  • Message de marque dilué
  • SEO divisé entre 2 pages
  • Funnel confus
  • 2 efforts marketing

B. PRSTO devient 100% Recruteur

Fermer /prsto candidat, tout basculer en B2B

✅ Pour
  • Message unique & puissant
  • SEO concentré
  • MRR élevé (99-349$/mois)
  • Positionnement défendable
  • Moins de churn que B2C
❌ Contre
  • Perd l'audience candidate
  • Marque à repositionner
  • Trafic organique perdu

C. Dual branding

PRSTO Candidat + PRSTO Pro (sous-marques)

✅ Pour
  • 2 audiences couvertes
  • Marque forte des deux côtés
  • Cross-sell possible
❌ Contre
  • Complexité produit ×2
  • Codebase plus lourde
  • 2 funnels à nourrir

D. Marketplace Recruteur → Candidat

Plateforme qui connecte recruteurs et candidats préparés

✅ Pour
  • Effet réseau
  • Positionnement unique
  • Lock-in des deux côtés
❌ Contre
  • Complexité énorme
  • Œuf/poule à résoudre
  • Modération nécessaire
  • Développement ×5
✦ Décision

3. Recommandation : Option B — PRSTO Réseau Recruteur

Transformer PRSTO en un réseau B2B où les recruteurs utilisent les outils PRSTO pour préparer et placer leurs candidats.

AVANT · Candidat

  • Hero : "L'OS qui décroche des postes de direction"
  • Cible : cadres dirigeants en recherche
  • Prix : 9.90€/semaine → 149.90€/6mois
  • Message : "reprenez le contrôle de votre carrière"
  • Produit one-shot : Brief 29.90€, Panel 14.90€
  • Limite : marché TAM de cadres dirigeants

APRÈS · Réseau Recruteur

  • Hero : "Le réseau qui prépare et connecte les talents"
  • Cible : recruteurs, cabinets RH, agences de placement
  • Prix : 99$/mois → 349$/mois (MRR ×3 à ×10 vs B2C)
  • Message : "préparez, placez, développez"
  • Produit : abonnement SaaS récurrent
  • Avantage : ARR prévisible, churn plus bas, TAM plus large

Argumentaire complet

💰 Business Model

  • MRR x3 à x10 vs B2C (99-349$ vs 9.90-39.90€)
  • Churn B2B naturellement plus bas (moyenne 3-5%/mois vs 5-10% B2C)
  • Contrats annuels possibles (paiement upfront)
  • Upsell facile : Solo → Pro → Elite → Sur-mesure
  • ROI immédiat : 1 placement = outil rentable 20x

🎯 Positionnement

  • Marché plus large : cabinets RH, recruteurs indep, agences, services RH internes
  • "Moteur de préparation de candidats" — catégorie vierge
  • Aucun ATS ne fait ce que PRSTO fait
  • Différenciation forte vs CV Design, TopCV, etc.
  • Complémentaire des ATS (Manatal, Recruit CRM, Bullhorn)

🚀 Produit

  • Workflow clair : 1 candidat + 1 offre = dossier complet en 8 min
  • Fonctionnalités déjà 90% prêtes (les outils candidats = outils recruteur)
  • Pas de nouveau code majeur : renommer + repositionner
  • Stats existantes (15-20h économisées, +40% placements, 92% ATS)
  • Comparatif déjà construit (PRSTO Candidat vs Recruteur vs ATS)

⚡ Execution rapide

  • Header : changer lien "Professionnel →" en "Candidat →"
  • /prsto → devient la page recruteur (renommer Hero + Features + Pricing)
  • /recruiters → redirige vers /prsto ou devient page secondaire
  • Aucun refacto lourd : les composants existent déjà des deux côtés
  • Migration en 1-2 jours de développement

6. Plan d'implémentation

Comment passer de l'état actuel à l'état cible, en conservant la qualité et sans réécriture lourde.

Phase 1 — Foundation (Jour 1-2)

  • Choisir le composant Hero définitif (fusionner HeroSection + RecruitersHero)
  • Uniformiser le header : remplacer "Professionnel →" par "Candidat →"
  • Rediriger /recruiters → /prsto (301 permanent)
  • Fusionner les sections Features (garder les 6 outils recruteur)
  • Fusionner les sections Pricing (garder la grille B2B 99-349$)

Phase 2 — Contenu (Jour 2-3)

  • Réécrire les textes du Hero pour le positionnement recruteur
  • Adapter tous les micro-textes : CTA, labels, descriptions
  • Mettre à jour le comparatif : PRSTO vs ATS traditionnels
  • Remplacer les témoignages candidats → témoignages recruteurs
  • Mettre à jour la FAQ pour les questions recruteur

Phase 3 — Dashboard (Jour 3-4)

  • Vérifier que le dashboard /recruiter/dashboard fonctionne
  • Adapter le workflow : 4 étapes au lieu de 3
  • Tester le funnel complet : landing → essai → dashboard
  • Configurer les emails de bienvenue recruteur

Phase 4 — Marketing (Jour 4-5)

  • Mettre à jour les meta tags SEO (title, description, OG)
  • Soumettre la nouvelle page à Google Search Console
  • Mettre à jour les ads / campagnes en cours
  • Préparer l'email annonce aux utilisateurs existants
  • Mettre à jour le branding si nécessaire

⚡ Ce qui ne change PAS

Identique ou presque
  • ✓ Le code existant des composants recruteur
  • ✓ Le dashboard recruteur (/recruiter/dashboard)
  • ✓ Les 6 fonctionnalités (mêmes outils, renommés)
  • ✓ Le système de paiement / stripe
  • ✓ Le design system (couleurs, typo, espacements)
Change
  • → Le Hero / message principal
  • → La navigation du header
  • → Le pricing (B2C → B2B)
  • → Les textes et témoignages
  • → Le SEO / meta tags
